输入与输出

在 Java 中，输入和输出（I/O）可以通过不同的方式处理。最常见的方式是使用 System.out 和 BufferedReader。

System.out 输出

这是 Java 中最基本的输出方式：

System.out.println 用于打印一行文本并换行。

System.out.print 也可以用来输出，但不会换行。

BufferedReader 输入

当你需要从控制台读取输入时，可以使用 BufferedReader，它比 Scanner 更高效，尤其在处理大量数据时。

BufferedReader 通过 readLine() 方法读取一行输入。

Integer.parseInt 用于将字符串转换成整数。

基本数据类型与控制流

基本数据类型

Java 的基本数据类型包括：

  • int、float、double、char、boolean 等等。

与 C 相比，Java 的基本类型是严格的，不能随意转换类型。

条件判断与循环

Java 中的条件判断与循环语法与 C 十分相似：

  • if、else、for、while 控制流语法与 C 几乎一样。
  • Java 中没有 do-while 语句，但你可以使用 while 模拟。

数组在 Java 中是对象，长度在创建后固定，访问时通过 arr[index]，长度使用 arr.length（注意它是字段，不是方法）。

面向对象编程基础

Java 是一门以面向对象为核心的编程语言。面向对象可以理解为：把程序中的数据和功能组织成一个个“对象”，每个对象都有自己的属性和行为。

类与对象

类是对象的模板，对象是类创建出来的具体实例。

Student 是一个类，用来描述学生这种对象。

name、age、score 是学生对象的属性。

创建对象时使用 new：

可以这样理解：

  • 类：模板，例如 Student
  • 对象：具体实例，例如 s
  • 属性：对象保存的数据
  • 方法：对象能执行的操作

成员变量与方法

成员变量写在类中，用来保存对象的状态。

方法写在类中，用来描述对象的行为。

name、age、score 是成员变量。

displayStudentInfo() 是方法。

构造方法

构造方法用于创建对象时初始化属性。构造方法的名字必须和类名完全一样，并且没有返回值类型。

创建对象时会自动调用构造方法：

this 关键字

this 表示当前对象。

当成员变量和参数同名时，常用 this 区分：

this.name 表示当前对象的成员变量。

右边的 name 表示传入的参数。

封装

封装是面向对象的重要思想。它的核心是：把对象内部的数据保护起来，不让外部随意访问或修改。

常见写法：

  • 成员变量使用 private
  • 对外提供 public 方法访问或修改数据

这样可以在修改数据时加入判断，防止非法数据进入对象。

Getter 和 Setter

Getter 用于获取属性，Setter 用于修改属性。

常见命名规则：

  • 获取 name：getName()
  • 修改 name：setName(String name)
  • 获取 age：getAge()
  • 修改 age：setAge(int age)

如果属性是 boolean 类型，Getter 常写成 isXxx()：

访问修饰符

Java 中常见的访问修饰符有：

  • public：公开，其他类可以访问
  • private：私有，只能在当前类内部访问
  • protected：受保护，当前类、子类和同包类可以访问

初学时可以先记住：

  • 属性通常用 private
  • 希望外部调用的方法用 public
  • 父类希望子类使用的成员可以用 protected

有些类主要表示数据，有些类主要负责管理数据。

例如：

Student 表示学生对象。

StudentManager 用来管理多个学生对象。

这样可以让代码职责更清楚：

  • 数据类负责保存信息
  • 管理类负责添加、删除、查找等操作

继承表示一个类可以复用另一个类的属性和方法。

Java 中一个类只能继承一个父类。

super 关键字

super 表示父类对象。常用于调用父类构造方法。

这表示先调用父类的构造方法，初始化父类中的属性。

方法重写

方法重写是指子类重新实现父类中的方法。

@Override 表示该方法重写了父类方法。

它可以帮助编译器检查方法名和参数是否正确，推荐写上。

抽象类

抽象类使用 abstract 修饰，不能直接创建对象，通常作为父类使用。

抽象方法只有声明，没有具体实现：

具体实现由子类完成。

抽象类适合表示一类事物的共同模板。

接口

接口使用 interface 定义。接口表示一种能力或规范。

类通过 implements 实现接口：

接口的特点是：谁实现了接口，谁就必须实现接口中定义的方法。

一个类只能继承一个父类，但可以实现多个接口。

继承和接口的区别

继承强调“是什么”。

可以理解为：信用卡是一种支付方式。

接口强调“能做什么”。

可以理解为：信用卡具有退款能力。

多态

多态表示：同一个父类引用，可以指向不同子类对象；同一个方法调用，会根据实际对象执行不同实现。

虽然变量类型都是 PaymentMethod，但实际执行的是不同子类中的 pay() 方法。

多态的好处是：可以面向父类编程，不必把代码写死在某一个具体类上。

调用时可以传入不同子类对象：

instanceof

instanceof 用来判断一个对象是否属于某个类，或者是否实现了某个接口。

在调用某些特殊方法前，可以先用 instanceof 判断对象是否具有对应能力。

static 关键字

static 表示成员属于类本身，而不是某一个具体对象。

最常见的是 main 方法：

普通成员变量属于对象，每个对象各有一份。

静态变量属于类，所有对象共享一份。

初学时不要滥用 static。如果一个属性应该属于每个对象自己，就不要写成 static。

final 关键字

final 表示不能再改变。

常见用法：

  • final 变量：不能重新赋值
  • final 方法：不能被子类重写
  • final 类：不能被继承

初学时最常见的是用 final 定义常量。

null

null 表示没有指向任何对象。

如果对 null 调用方法，会出现 NullPointerException。

面向对象三大特征

面向对象常说有三大特征：

  • 封装：把数据保护起来，通过方法控制访问
  • 继承：子类复用父类已有内容
  • 多态：同一种调用方式，根据实际对象执行不同实现

这三点是理解 Java 面向对象的核心。

常用类库与 API

Java 提供了丰富的标准类库。下面按正文的顺序整理一些最常用的类和方法，方便你对照后面的示例一起复习：

  • 字符串处理：String、StringBuilder、StringBuffer，常用于拼接、截取、替换和比较字符串
  • 集合框架：ArrayList、HashMap、LinkedHashMap、HashSet、TreeMap，常用于存储和组织数据
  • 文件处理：File、BufferedReader、BufferedWriter、FileReader、FileWriter，常用于文件读写和路径操作
  • 栈与队列：Stack、ArrayDeque，常用于表达式求值、括号匹配和滑动窗口
  • 工具类：Collections、Math、Character、Integer，常用于排序、数学运算、字符判断和类型转换
  • 日期与时间：LocalDate、LocalTime、LocalDateTime，常用于获取、计算和格式化时间

String

String 是 Java 中最常用的类之一，用于表示不可变字符串。

split(String regex, int limit)：按规则切分字符串

StringBuilder 和 StringBuffer

可变字符串，常用于拼接。StringBuffer 线程安全，StringBuilder 更高效。

append(String str)：追加字符串

setCharAt(int index, char ch)：修改指定位置字符

集合框架类

List（ArrayList, LinkedList）

有序集合，允许重复元素。

ArrayList 基于动态数组，适合随机访问；LinkedList 基于链表，适合频繁插入和删除。

前提代码：

add(E e)：添加元素

get(int index)：获取元素

size()：元素数量

contains(Object o)：是否包含元素

remove(int index)：删除元素

set(int index, E element)：修改指定位置元素

isEmpty()：判断集合是否为空

Set（HashSet, TreeSet）

无序且不重复元素集合。

HashSet 基于哈希表，查询速度快；TreeSet 基于红黑树，元素会按排序规则保持有序。

前提代码：

add(E e)：添加元素

contains(Object o)：是否包含元素

remove(Object o)：删除元素

Map（HashMap, LinkedHashMap, TreeMap）

键值对集合，键唯一。

HashMap 基于哈希表，适合快速查找；TreeMap 基于红黑树，键会按顺序排列。

前提代码：

put(K key, V value)：添加键值对

get(Object key)：获取值

containsKey(Object key)：是否包含键

remove(Object key)：删除键值对

putIfAbsent(K key, V value)：键不存在时才放入

getOrDefault(Object key, V defaultValue)：取值并设置默认值

merge(K key, V value, BiFunction remappingFunction)：合并更新（计数场景很常见）

keySet()：获取键集合

values()：获取值集合
